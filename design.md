
# Mapingo - Design Guidelines

## Brand Name

Mapingo

## Slogan

"Learn the states of Mexico by playing."

## Brand Personality

Mapingo should feel like a friendly learning companion for students.

The design should be:

- Playful
- Colorful
- Rounded
- Clean
- Cheerful
- Motivating
- Safe for children
- Easy to understand

The app should take inspiration from gamified learning apps, but it must have its own original visual identity.

## Visual Direction

Mapingo should combine:

- Educational clarity
- Game-like motivation
- Mexican geography theme
- Friendly illustrations
- Simple navigation
- Soft animations

The interface should avoid feeling like a traditional school app. It should feel more like a learning game.

## Color Palette

### Primary Color

Use a bright green as the main brand color.

Purpose:

- Main buttons
- Progress
- Correct answers
- Lesson path
- Positive feedback

Suggested color:

```txt
Primary Green: #58CC02
````

### Secondary Color

Use a friendly blue for maps, information, and secondary actions.

```txt
Map Blue: #1CB0F6
```

### Accent Color

Use yellow or orange for rewards and achievements.

```txt
Reward Yellow: #FFC800
Orange Accent: #FF9600
```

### Error Color

Use red only for mistakes or warnings.

```txt
Error Red: #FF4B4B
```

### Background Colors

Use soft backgrounds.

```txt
Light Background: #F7F9FB
Card Background: #FFFFFF
Soft Green Background: #E9FAD8
Soft Blue Background: #E8F6FF
```

### Text Colors

```txt
Primary Text: #2B2B2B
Secondary Text: #777777
Disabled Text: #B0B0B0
```

## Typography

Use a rounded, modern, friendly font.

Recommended fonts:

* Nunito
* Baloo 2
* Inter
* Poppins

Preferred option:

```txt
Nunito
```

Typography should be readable for children.

Suggested scale:

* Large title: 28px - 32px
* Screen title: 24px
* Section title: 18px - 20px
* Body text: 16px
* Button text: 16px - 18px
* Small labels: 12px - 14px

## UI Style

### Cards

Cards should be rounded and soft.

Recommended:

```txt
Border radius: 20px - 28px
Padding: 16px - 24px
Shadow: soft and subtle
```

### Buttons

Buttons should be large, rounded, and easy to tap.

Primary buttons:

* Green background
* White bold text
* Rounded corners
* Slight bottom shadow to feel clickable

Secondary buttons:

* White background
* Colored border
* Colored text

Disabled buttons:

* Light gray background
* Muted text

### Icons

Use simple, rounded icons.

Recommended icon style:

* Filled or rounded line icons
* Friendly and consistent
* Avoid overly detailed icons

### Animations

Animations should be short and rewarding.

Use animations for:

* Correct answers
* Wrong answers
* Lesson completion
* XP earned
* Streak increase
* Unlocking lessons

Animations should not slow down the experience.

## Mascot

Mapingo should have an original mascot.

Recommended mascot concept:

A friendly axolotl explorer.

Possible traits:

* Curious
* Cheerful
* Encouraging
* Mexican identity
* Explorer hat or small backpack
* Map or compass accessory

The mascot should appear in:

* Onboarding
* Feedback messages
* Lesson completion
* Streak screen
* Empty states

Example messages:

* "Great job!"
* "You found it!"
* "Almost there!"
* "Let's try again!"
* "You are becoming a Mexico map expert!"

## App Navigation

Use simple bottom navigation with 3 to 4 tabs.

Recommended tabs:

1. Learn
2. Practice
3. Map
4. Profile

## Main Screens

### 1. Splash Screen

Should show:

* Mapingo logo
* Mascot
* Short loading animation

### 2. Onboarding Screens

Should include:

* Welcome message
* App purpose
* Daily goal selection
* Start learning button

Visual style:

* Large mascot illustration
* Large title
* Minimal text
* Strong call to action

### 3. Home / Learning Path

The home screen should show:

* Current unit
* Learning path
* Lesson nodes
* Locked and unlocked lessons
* XP summary
* Streak indicator

Lesson nodes:

* Circular
* Green when completed
* White/outlined when available
* Gray when locked
* Star or crown for unit review

### 4. Lesson Screen

The lesson screen should include:

* Progress bar at the top
* Hearts or mistake count if used
* Question title
* Exercise content
* Answer options
* Check button
* Feedback area

The user should not feel overwhelmed.

Only show one question at a time.

### 5. Map Game Screen

The map screen should include:

* Question at the top
* Interactive map of Mexico
* Zoom or pan if needed
* Tap feedback
* Continue button after answer

Map states should visually highlight on interaction.

### 6. Match Game Screen

The match game should show:

* State cards on one side
* Capital cards on the other side
* Tap-to-match interaction
* Correct pairs should disappear or become locked

### 7. Lesson Result Screen

After a lesson, show:

* XP earned
* Accuracy percentage
* Correct answers
* Mistakes
* Streak update
* Continue button

Make this screen feel rewarding.

### 8. Practice Screen

The practice screen should include:

* Review mistakes
* Practice capitals
* Practice map
* Quick challenge
* Random quiz

### 9. Map Explorer Screen

The map explorer should let users explore Mexico freely.

When tapping a state, show:

* State name
* Capital
* Region
* Fun fact
* Progress status

### 10. Profile Screen

The profile screen should show:

* User avatar or mascot
* User name
* Total XP
* Daily streak
* Completed units
* Achievements
* Accuracy

## UX Principles

### 1. Short Sessions

Lessons should be short.

A user should be able to complete a lesson in 3 to 5 minutes.

### 2. Immediate Feedback

Every answer should produce immediate visual and emotional feedback.

Correct:

* Green highlight
* Positive sound or animation
* Encouraging message

Wrong:

* Red highlight
* Show correct answer
* Friendly explanation
* No harsh punishment

### 3. Repetition Without Boredom

The same content should appear in different exercise formats.

Example:

* State name to capital
* Capital to state
* Map location
* Silhouette recognition
* Fun fact recognition

### 4. Safe for Students

Avoid:

* Ads in the MVP
* Social chat
* External links
* Public profiles
* Inappropriate content
* Competitive pressure

### 5. Accessibility

The app should support:

* Large tap targets
* High contrast
* Clear readable fonts
* Text labels with icons
* Avoid color-only feedback

## Component Guidelines

### Primary Button

Usage:

* Continue
* Start lesson
* Check answer
* Complete onboarding

Style:

* Green background
* White bold text
* Rounded corners
* Full width when possible

### Answer Card

Usage:

* Multiple choice answers

Style:

* White card
* Rounded corners
* Subtle border
* Large text
* Selected state with blue border
* Correct state with green border
* Wrong state with red border

### Lesson Node

Usage:

* Learning path

States:

* Completed
* Current
* Locked
* Review

### Progress Bar

Usage:

* Lesson progress
* Unit progress

Style:

* Rounded
* Green fill
* Soft background

## Tone of Voice

The app copy should be short, positive, and friendly.

Use messages like:

* "Nice work!"
* "You got it!"
* "Keep going!"
* "Let's try one more!"
* "Great progress!"
* "Practice makes it easier!"

Avoid:

* "Wrong!"
* "You failed!"
* "Bad answer!"
* "Try harder!"

Use instead:

* "Almost!"
* "Good try!"
* "The correct answer is..."
* "Let's practice this again later."

## Design Priority for MVP

The first version should prioritize:

1. Clear learning flow.
2. Fun interactions.
3. Fast feedback.
4. Easy navigation.
5. Clean visual style.
6. Simple gamification.

The app should be beautiful, but functionality and learning value are more important than complex animations.
